The Business Challenge

⚠️ Legacy Ruby Limitations
  • Mixed REST/GraphQL causing code redundancy
  • Duplicate permission logic across endpoints
  • ActiveRecord row/column limitations
  • No caching for static data
  • Cannot exceed file size limits
👥 Implementation Team Pain
  • Loading settings page multiple times daily
  • 8-12 second wait per page load
  • Downloading unnecessary data
  • Permission checks in every module
  • Productivity bottleneck

Core Issue: Ruby's architectural limitations preventing efficient scaling as Cast & Crew consolidates platforms

Technical Implementation

🏗️ Consolidated Architecture

• Unified REST endpoints → Single GraphQL resolver

• Centralized permission checking at login (admin/non-admin)

• Eliminated redundant offer/permission checks across modules

• Used AI tools for Ruby → .NET code conversion

Better Migrations: Superior to Ruby ActiveRecord

No Limitations: No row/column restrictions like Ruby
